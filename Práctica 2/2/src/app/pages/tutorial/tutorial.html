Creación y estructura básica de un proyecto Angular

En esta sección se explica cómo crear un nuevo proyecto con Angular CLI, qué opciones aparecen al ejecutar el comando ng new y cómo se ha organizado esta práctica para convertirla en una aplicación de tipo SPA (Single Page Application) con varias páginas: Inicio, Instalación, Tutorial y Funcionamiento.

1. Crear un nuevo proyecto con Angular CLI

Una vez instalado el entorno, el primer paso para comenzar a trabajar con Angular es crear un proyecto nuevo. Para ello, se utiliza el siguiente comando desde la terminal:

ng new practica2-dsanchez


El nombre del proyecto puede ser cualquiera (por ejemplo proyecto-prueba), pero en esta práctica se ha utilizado practica2-dsanchez.


Al ejecutar este comando, Angular CLI hace varias preguntas:

  • ¿Quieres añadir routing (enrutamiento)?
    En esta práctica se ha elegido Yes, para poder crear una SPA con varias páginas (Inicio, Instalación, Tutorial, Funcionamiento).
  • ¿Qué tipo de hojas de estilo deseas utilizar?
    Se ha seleccionado CSS, aunque también se podrían usar otras sin ningun tipo de problema.
  • ¿Enable Server-Side Rendering (SSR) / SSG?
    Para un proyecto básico y de aprendizaje se ha respondido No, ya que añade complejidad innecesaria.

Tras responder a estas preguntas, Angular CLI genera automáticamente la estructura inicial del proyecto.

2. Estructura básica de un proyecto Angular

Una vez creado el proyecto, la estructura de carpetas más importante es la siguiente:

  • src/: carpeta principal del código fuente de la aplicación.
  • src/app/: contiene la lógica principal de la aplicación (componentes, rutas, etc.).
  • src/app/app.ts: componente raíz de la aplicación (App), donde se define el selector, la plantilla principal (app.html) y los estilos (app.css).
  • src/app/app.routes.ts: archivo donde se configuran las rutas de la aplicación para poder navegar entre las distintas páginas sin recargar el navegador.
  • src/styles.css: hoja de estilos globales que afecta a toda la aplicación.
  • src/main.ts: punto de entrada donde se arranca la aplicación Angular.

Esta estructura se genera automáticamente con ng new, por lo que el desarrollador puede empezar a trabajar directamente sin tener que crear estos archivos a mano.

3. Ejecutar el proyecto por primera vez

Una vez generado el proyecto, el siguiente paso es ejecutarlo para comprobar que todo funciona correctamente. Para ello, se accede a la carpeta del proyecto y se lanza el servidor de desarrollo:

cd practica2-dsanchez

ng serve --open


El parámetro --open hace que el navegador se abra automáticamente en la dirección http://localhost:4200. Si todo está correcto, se mostrará la página inicial de Angular.

4. Creación de las páginas del tutorial

Para esta práctica se han creado varias páginas (componentes) utilizando Angular CLI, de forma que cada una corresponde a un apartado del tutorial:

ng generate component pages/inicio --standalone

ng generate component pages/instalacion --standalone

ng generate component pages/tutorial --standalone

ng generate component pages/funcionamiento --standalone


Cada uno de estos comandos crea un componente standalone dentro de la carpeta src/app/pages/, con sus propios archivos de TypeScript, HTML y CSS.

En esta práctica, cada componente representa una sección del tutorial: la página de Inicio, la instalación, la explicación general del proyecto (esta página de "Tutorial") y el funcionamiento del framework.
